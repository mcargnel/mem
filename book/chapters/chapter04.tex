Tal como se mencionó anteriormente, uno de los \textit{trade-off} de los ensambles de árboles es que al agrupar múltiples árboles el modelo en conjunto deja de ser interpretable. Esto puede no suponer un problema si el objetivo es predecir, pero si el objetivo es interpretar el modelo el hecho de trabajar con 'cajas negras' puede ser un problema.

Es por ello que se desarrollaron técnicas para extraer información de modelos de aprendizaje automático. Estas técnicas se aplican una vez entrenado el modelo y permiten entender cómo el mismo utiliza las variables. Pese a que existen distintas técnicas para interpretar modelos, algunas son específicas para ciertos tipos de modelos mientras que otras son agnósticas al modelo. En este trabajo se estudiará como acceder a la importancia de cada covariable y al efecto que tiene cada una de ellas sobre la variable dependiente.

\section{Importancia de las variables}

Con el acceso a información que se tiene actualmente, no es extraño querer incluir múltiples covariables en un mismo modelo. Sin embargo, entender cuáles de ellas realmente aportan al mismo no es trivial, por lo que se desarrollaron múltiples técnicas para evaluarlo. Esta sección presenta dos enfoques fundamentales: \textit{Split Gain Feature Importance} y \textit{Permutation Feature Importance}. Posteriormente, se analiza una limitación importante de estas técnicas (la inestabilidad de las importancias entre modelos con rendimiento similar) y se presenta \textit{Model Class Reliance} como una solución a este problema.

\subsection{\textit{Split Gain Feature Importance}}

Esta técnica fue propuesta por \cite{friedman_2001} y asigna importancias en base a la reducción del error que cada variable genera al ser utilizada como criterio de división. Esta metodología es específica para árboles de decisión y se define formalmente para un solo árbol como:
 
\begin{equation}
    \label{eq:split-importance}
\hat{I}_j^{2}(T)=\sum_{t=1}^{N-1}\hat{i}_t^{2}1(v_t=j)    
\end{equation}

donde $\hat{I}_j^{2}(T)$ representa la importancia de la variable $j$ en el árbol $T$, $\hat{i}_t^{2}$ es la mejora en el criterio de división del nodo $t$, y $v_t$ indica la variable utilizada para dividir dicho nodo. La suma se realiza sobre todos los nodos no terminales.

Cuando se trabaja con ensambles de árboles, la importancia se generaliza promediando sobre todos los árboles del ensamble:

\begin{equation}
\hat{\mathbf{I}}_j^{2}=\frac{1}{B}\sum_{m=1}^{B}\hat{I}^{2}_j(T_b)    
\end{equation}

Esta medida presenta limitaciones importantes. Primero, está basada en heurísticas sin fundamentos estadísticos sólidos. \cite{strobl_2008} critica que otorga demasiada influencia a variables correlacionadas. Segundo, favorece a variables categóricas con muchas categorías, dado que estas son propensas a generar más divisiones sin que esto implique una mayor contribución a la reducción del error.

\subsection{\textit{Permutation Feature Importance}} 

Una alternativa es \textit{Permutation Feature Importance} (PFI), técnica desarrollada por \cite{breiman2001a} para \textit{Random Forest} y posteriormente generalizada para cualquier modelo por \cite{fischer_2019}. La intuición detrás de esta técnica es medir cómo cambia el error del modelo cuando se destruye la relación entre una variable y la respuesta mediante una permutación aleatoria. Las variables más importantes serán aquellas cuya permutación genere mayor incremento en el error.

En el contexto de \textit{Random Forest}, habiendo entrenado un modelo $\hat{f}$, para cada covariable $j \in \{1,2,...,p\}$ se construye un nuevo conjunto de datos permutando las observaciones de dicha variable mientras se mantienen las demás fijas. Luego se calcula el error \textit{out-of-bag}:

\begin{equation}
E_{OOB}(X_j) = \frac{1}{n} \sum_{i=1}^{n} \left( y_i -  \hat{f}^{(b)}(X_{i,j}) \right)^2    
\end{equation}

La importancia se obtiene comparando este error con el del modelo original:
\begin{equation}
    PFI(j) = E_{OOB}(X_j) - E_{OOB}(X) ~~ \text{ó} ~~ PFI(j) = \frac{E_{OOB}(X_j)}{E_{OOB}(X)}
\end{equation}

\subsubsection{Generalización: \textit{Model Reliance}}

La formulación original de PFI depende del error \textit{out-of-bag}, lo que limita su aplicabilidad a modelos basados en \textit{bagging}. \cite{fischer_2019} proponen una generalización denominada \textit{Model Reliance} (MR) que es aplicable a cualquier modelo.

Partiendo de un modelo entrenado $\hat{f}$, una variable de respuesta $y$, una función de pérdida $L$ y una matriz de diseño $X$, para cada covariable $j$ se permutan sus valores y se calcula la pérdida esperada:

\begin{equation}
L_{perm}(f)= \mathcal{E}[L(f,(Y,X_1^{perm},X_2))]    
\end{equation}

donde $X_1^{perm}$ representa la variable permutada y $X_2$ las demás variables que permanecen constantes. La importancia se define como el cociente entre la pérdida con permutación y la pérdida original:

\begin{equation}
MR^{j}(f)=\frac{L^{j}_{perm}(f)}{L(f)}    
\end{equation}

Para estimar $L_{perm}$ en la práctica, los autores proponen dos alternativas. La primera realiza todas las permutaciones posibles:

\begin{equation}
    \hat{e}_{perm}(F) = \frac{1}{n(n-1)}\sum_{i=1}^n\sum_{j\neq i}L\{f,y_j,X_{1[i,.]},X_{2[i,.]} \}
\end{equation}

Dado que esto puede ser computacionalmente intensivo, proponen como alternativa dividir la muestra a la mitad e intercambiar los valores de la variable de interés entre ambas mitades:

\begin{equation}
    \begin{aligned}
    \hat{e}_{\text{divide}}(f) &= \frac{1}{2 \lfloor n/2 \rfloor} \sum_{i=1}^{\lfloor n/2 \rfloor} \left[
    L\Bigl(f,\Bigl(y_{[i]},\, X_{1\bigl[i+\lfloor n/2 \rfloor\bigr]},\, X_{2[i]}\Bigr)\Bigr) \right. \\
    &\left. \quad + L\Bigl(f,\Bigl(y_{i+\lfloor n/2 \rfloor},\, X_{1[i]},\, X_{2\bigl[i+\lfloor n/2 \rfloor\bigr]}\Bigr)\Bigr)
    \right]. \\
    \end{aligned}    
\end{equation}

Este fenómeno se conoce como el \textit{Rashomon Effect} \cite{breiman_2001}, término inspirado en la película de Kurosawa donde un mismo evento es narrado desde perspectivas contradictorias. \cite{fischer_2019} formalizan este concepto definiendo el ``conjunto de Rashomon'' como la colección de modelos con rendimiento predictivo similar pero estructuras internas diferentes. La existencia de múltiples explicaciones igualmente válidas representa un desafío fundamental: la importancia de una variable no es una propiedad intrínseca de los datos, sino que depende del modelo específico elegido.

\subsection{\textit{Model Class Reliance}: una solución al \textit{Rashomon Effect}} 

Para abordar la inestabilidad de las importancias, \cite{fischer_2019} proponen \textit{Model Class Reliance} (MCR). En lugar de reportar la importancia de un único modelo, MCR calcula el rango de importancias posibles sobre todos los modelos con rendimiento similar al óptimo.

Primero se define formalmente el conjunto de Rashomon:

\begin{equation}
R(\epsilon) = \{f \in \mathcal{F} \mid e_{orig}(f) \le e_{orig}(f_{ref}) + \epsilon \}    
\end{equation}

Este conjunto contiene todos los modelos de la clase $\mathcal{F}$ cuya pérdida sea como máximo $\epsilon$ mayor que la del modelo de referencia $f_{ref}$ que minimiza el error.

Luego, el intervalo MCR se define como:

\begin{equation}
[MCR^{-}(\epsilon), MCR^{+}(\epsilon)]=[\min_{f\in R(\epsilon)}MR(f), \max_{f\in R(\epsilon)}MR(f)]    
\end{equation}

\subsubsection{Interpretación}
El intervalo MCR proporciona información sobre la consistencia de la importancia de una variable. Si $MCR^{-}(\epsilon) > 1$, todos los modelos dentro del conjunto de Rashomon dependen de la variable, indicando que es consistentemente importante. Si $MCR^{+}(\epsilon) \approx 1$, ningún modelo depende fuertemente de ella.

La amplitud del intervalo también es informativa: un intervalo estrecho indica que la importancia es estable independientemente del modelo elegido, mientras que un intervalo amplio señala que la importancia depende fuertemente de la arquitectura particular. Por ejemplo, si para una variable $j$ se tiene $MCR^{-}(\epsilon) \approx 1$ pero $MCR^{+}(\epsilon) >> 1$, existen modelos con buena capacidad predictiva que no dependen de dicha variable, por lo que su importancia es cuestionable.

\subsubsection{Estimación}
En la práctica, MCR se estima mediante un estimador \textit{plug-in}: se calcula $\hat{MR}(f)$ para todos los modelos que integren el conjunto de Rashomon empírico $\hat{R}(\epsilon)$, y se reportan los valores mínimo y máximo como límites del intervalo.

\section{Efecto de las variables} 

Mientras que las técnicas de importancia de variables permiten identificar qué covariables son relevantes para el modelo, no proporcionan información sobre la forma de la relación entre cada variable y la respuesta. Para ello se desarrollaron técnicas de visualización que permiten aislar y representar el efecto de una o más covariables sobre la predicción del modelo.

Esta sección presenta tres técnicas complementarias: los \textit{Partial Dependence Plots} (PDP), que estiman el efecto marginal promedio de una variable; los \textit{Individual Conditional Expectation} (ICE), que permiten visualizar heterogeneidad en los efectos; y las variantes centradas y derivadas que facilitan la identificación de interacciones.

\subsection{\textit{Partial Dependence Plots} (PDP)}

Los PDP fueron introducidos por \cite{friedman_2001} como una técnica para visualizar el efecto marginal de una o más covariables sobre la predicción del modelo. La idea central es aislar el efecto de un subconjunto de variables $X_S$ marginalizando sobre las demás variables $X_C$.

Formalmente, la dependencia parcial se define como:

\begin{equation}
    \label{eq:pdp}
\bar{f}(X_S)=\mathcal{E}_{X_C}[\hat{f}(x)]=\int \hat{f}(X_S,X_C)p(X_C)dX_C    
\end{equation}

donde $\hat{f}(x)$ es el modelo entrenado, $X_S$ es el subconjunto de variables cuyo efecto se desea estimar, $X_C$ representa las variables complementarias, $p(X_C)$ es su distribución marginal, y $\mathcal{E}_{X_C}[\cdot]$ denota la esperanza sobre dicha distribución.

En la práctica, la distribución $p(X_C)$ es desconocida y se aproxima mediante la distribución empírica de los datos de entrenamiento:
 
\begin{equation}
\bar{f}(X_S)\approx\frac{1}{n}\sum_{i=1}^{n}\hat{f}(X_S,x_{C}^{i})
\end{equation}

El algoritmo para calcular el PDP procede de la siguiente manera: para cada valor de interés $x_S$ de la variable a analizar, se construyen $n$ observaciones artificiales combinando $x_S$ con los valores observados $x_C^{(i)}$ de las demás variables para cada observación $i$ en la muestra. El modelo genera predicciones $\hat{f}(x_S, x_C^{(i)})$ para cada una de estas combinaciones, y el valor del PDP en $x_S$ es el promedio de estas $n$ predicciones. Este procedimiento se repite para una grilla de valores de $x_S$, generando la curva de dependencia parcial.

Esta metodología tiene limitaciones importantes. El supuesto implícito es que las variables $X_S$ son independientes de $X_C$, lo cual raramente se cumple en la práctica. Cuando existen relaciones entre variables, el PDP puede generar predicciones para combinaciones de valores que no existen en los datos reales. Por ejemplo, si se analiza el efecto de la altura sobre el peso predicho, el PDP evaluará combinaciones donde una persona mide 2 metros pero pesa 50 kg, una combinación improbable que puede distorsionar la interpretación.

Para ilustrar los PDP se utilizó el mismo modelo de \textit{Gradient Boosting Machine} entrenado anteriormente sobre el conjunto de datos \textit{Abalone}. La \autoref{fig:partial-dependence-plot} muestra el efecto del peso total sobre la edad predicha. Se observa una relación positiva: a mayor peso, mayor edad predicha, aunque la relación se aplana para pesos elevados.

\subsection{\textit{Individual Conditional Expectation} (ICE)}

Los PDP muestran el efecto promedio de una variable, pero este promedio puede ocultar heterogeneidad importante. \cite{Goldstein2015} propusieron los gráficos ICE como una extensión de los PDP que permite visualizar el efecto de una variable para cada observación individual.

Formalmente, para cada observación $i$ se define:

\begin{equation}
\hat{f}_i(x_S)=\hat{f}(x_S, x_{C}^{(i)})    
\end{equation}

Es decir, en lugar de promediar sobre todas las observaciones, se mantienen fijos los valores $x_C^{(i)}$ de cada observación y se varía únicamente $x_S$. El resultado es una curva para cada observación en la muestra, lo que permite visualizar cómo varía el efecto de $X_S$ dependiendo de los valores de las otras variables.

La relación entre PDP e ICE es directa: el PDP es simplemente el promedio de todas las curvas ICE:

\begin{equation}
\bar{f}(x_S) = \frac{1}{n}\sum_{i=1}^{n}\hat{f}_i(x_S)
\end{equation}

Cuando las variables son independientes y no existen interacciones, todas las curvas ICE deberían ser paralelas y el PDP las resume adecuadamente. Sin embargo, cuando las curvas ICE divergen significativamente, esto indica la presencia de interacciones entre $X_S$ y alguna de las variables en $X_C$. Por esta razón, se recomienda siempre visualizar las curvas ICE junto con el PDP para detectar posibles heterogeneidades.

La \autoref{fig:ice-plot} muestra las curvas ICE para la misma variable (peso total) analizada anteriormente. Se observa que, aunque la tendencia general es positiva, existe considerable variación entre observaciones. Esta dispersión sugiere que el efecto del peso sobre la edad predicha depende de los valores de otras características del organismo.

\subsubsection{\textit{Centered ICE} (c-ICE)}

Un problema práctico de los gráficos ICE es que las curvas tienen diferentes ordenadas al origen, lo que dificulta la comparación visual y la identificación de interacciones. Para abordar este problema, \cite{Goldstein2015} proponen los gráficos ICE centrados, donde cada curva se centra en un punto de referencia común.

Formalmente, se elige un valor de referencia $x^*$ (típicamente el mínimo o máximo de $X_S$) y se centra cada curva restando su valor en dicho punto:

\begin{equation}
\hat{f}_{cent}^{(i)}(x_S) = \hat{f}(x_S, x_{C}^{(i)}) - \hat{f}(x^*, x_{C}^{(i)})
\end{equation}

De esta manera, todas las curvas pasan por el origen en $x^*$. Si se elige el valor mínimo como referencia, las curvas muestran el efecto acumulado de incrementar $X_S$ desde su valor mínimo. Esta representación facilita la identificación de interacciones: si las curvas centradas son paralelas, el efecto de $X_S$ es homogéneo; si divergen, existen interacciones con otras variables.

La \autoref{fig:centered-ice-plot} presenta el gráfico ICE centrado para el peso total, utilizando el valor mínimo como referencia. Se pueden observar diferencias en las pendientes de las curvas, lo que confirma la presencia de heterogeneidad en el efecto. La línea roja representa el PDP centrado, que resume el efecto promedio.

\subsubsection{\textit{Derivative ICE} (d-ICE)}

Como complemento a los gráficos centrados, \cite{Goldstein2015} proponen los gráficos de la derivada de las curvas ICE. Este enfoque se basa en la observación de que, si no existen interacciones entre $X_S$ y $X_C$, el modelo puede descomponerse aditivamente:

\begin{equation}
\hat{f}(x)=\hat{f}(x_S,x_C)=g(x_S)+h(x_C) \Rightarrow \frac{\partial \hat{f}(x)}{\partial x_S}=g'(x_S)    
\end{equation}

En este caso, la derivada de $\hat{f}$ con respecto a $X_S$ depende únicamente de $X_S$ y es idéntica para todas las observaciones. Por lo tanto, si las curvas derivadas coinciden, no existen interacciones; si difieren, la relación entre $X_S$ y la predicción depende de los valores de otras variables. Esta herramienta es particularmente útil cuando las curvas ICE centradas son difíciles de interpretar visualmente debido a la cantidad de observaciones o a la complejidad de los patrones.

\section{Extensiones}

Las técnicas de interpretación presentadas en las secciones anteriores permiten comprender cómo un modelo utiliza las variables para generar predicciones. Sin embargo, surgen naturalmente dos preguntas adicionales: ¿pueden estas técnicas informar sobre relaciones causales? y ¿qué relación tienen con el verdadero proceso generador de los datos? Esta sección aborda ambas cuestiones, explorando las condiciones bajo las cuales las técnicas de interpretación admiten una lectura causal y cómo se relacionan formalmente con las propiedades poblacionales.

\subsection{Interpretación causal de PDP e ICE}

Los modelos de aprendizaje automático convencionales se basan en asociaciones estadísticas y no contemplan intervenciones ni escenarios contrafactuales \cite{pearl_2018}. No obstante, resulta de interés analizar bajo qué condiciones las técnicas de interpretación pueden admitir una lectura causal. Este es el eje central del trabajo de \cite{Zhao_2021}, donde se demuestra que los PDP poseen una interpretación causal bajo supuestos específicos.

\subsubsection{Conexión con el \textit{backdoor adjustment}}

La clave de la interpretación causal reside en la conexión entre el PDP y el \textit{backdoor adjustment} de \cite{Pearl1993}. El ajuste \textit{backdoor} permite identificar el efecto causal de una variable $X_S$ sobre $Y$ cuando se dispone de un conjunto de variables $X_C$ que satisface ciertas condiciones. Formalmente:

\begin{equation}
    \label{eq:back-door}
P(Y|do(X_S=x_S))=\int P(Y|X_S=x_S, X_C=x_C)dP(x_C)    
\end{equation}

donde $P(Y|do(X_S=x_S))$ representa la distribución de $Y$ tras una intervención que fija $X_S$ en el valor $x_S$. Esta fórmula permite identificar el efecto causal de $X_S$ sobre $Y$ siempre que se cumplan dos condiciones: (1) ninguna variable en $X_C$ sea descendiente de $X_S$ en el grafo causal, y (2) $X_C$ bloquee todos los caminos \textit{backdoor} entre $X_S$ e $Y$, es decir, todos los caminos que conectan $X_S$ con $Y$ a través de causas comunes.

Para establecer la conexión con el PDP, se parte del modelo entrenado $\hat{f}(x)$ que aproxima la esperanza condicional:

\begin{equation}
    \label{eq:back-door-expectation}
    \hat{f}(x) \approx E[Y \mid X=x]
\end{equation}

Si el modelo está bien especificado, es decir, $\hat{f}(x_S, x_C) = E[Y \mid X_S = x_S, X_C = x_C]$, entonces el PDP puede expresarse como:

\begin{equation}
\begin{aligned}
\text{PDP}(x_S) &= \int \hat{f}(x_S, x_C) \, dP(x_C) \\
                &= \int E[Y \mid X_S = x_S, X_C = x_C] \, dP(x_C) \\
                &= E[Y \mid do(X_S = x_S)]
\end{aligned}
\end{equation}

Esta equivalencia muestra que el PDP coincide con el efecto causal intervencionista cuando: (1) el modelo aproxima correctamente la esperanza condicional, y (2) el conjunto $X_C$ satisface el criterio \textit{backdoor}. Bajo estas condiciones, el PDP puede interpretarse como el efecto promedio de intervenir en $X_S$, lo que proporciona una justificación causal para su uso.

\subsubsection{Métodos alternativos para inferencia causal}

Cuando las condiciones del criterio \textit{backdoor} no se satisfacen, o cuando el objetivo explícito es la inferencia causal, existen \textit{frameworks} diseñados específicamente para este propósito.

El \textit{Double Machine Learning} (DML) propuesto por \cite{Chernozhukov_2018} utiliza modelos de aprendizaje automático para estimar los \textit{nuisance parameters}: las relaciones entre el tratamiento y los \textit{confounders}, y entre el resultado y los \textit{confounders}. Mediante el uso de \textit{cross-fitting} (entrenamiento y predicción en muestras separadas), DML permite obtener estimadores del efecto causal con propiedades estadísticas deseables, incluyendo normalidad asintótica y validez de intervalos de confianza, incluso cuando se utilizan modelos complejos como ensambles de árboles o redes neuronales.

Los \textit{Causal Forests} de \cite{Wager_2018} extienden los \textit{Random Forests} para la estimación de efectos de tratamiento heterogéneos (CATE, por sus siglas en inglés). A diferencia de los \textit{Random Forest} tradicionales que optimizan la predicción del resultado observado, los \textit{Causal Forests} adaptan la función de división para maximizar la heterogeneidad de los efectos del tratamiento entre las particiones. Esto permite identificar subgrupos de la población con respuestas diferenciales a una intervención, capturando la estructura local de la heterogeneidad causal.

Estos métodos representan un cambio de paradigma respecto a las técnicas de interpretación clásicas: en lugar de interpretar un modelo predictivo, construyen modelos específicamente diseñados para estimar efectos causales. Por esta razón, su estudio en profundidad excede el alcance de este trabajo.

\subsection{Relación con el proceso generador de datos} 

Una diferencia fundamental entre los modelos estadísticos tradicionales y los modelos de aprendizaje automático es la interpretabilidad de los parámetros. En un modelo de regresión lineal, los coeficientes tienen una interpretación directa en términos del proceso generador de datos (DGP): representan el cambio esperado en $Y$ ante un cambio unitario en la variable correspondiente, manteniendo las demás constantes. Esta conexión permite realizar inferencia sobre propiedades poblacionales.

Los modelos de aprendizaje automático, en cambio, priorizan la capacidad predictiva sobre la interpretabilidad paramétrica. Las técnicas presentadas en este capítulo (PDP, PFI, ICE) fueron diseñadas para extraer información de estos modelos, pero surge la pregunta: ¿qué relación tienen estas medidas con las verdaderas propiedades del DGP?

\cite{freiesleben_2024} proponen un marco estadístico que formaliza esta conexión, introduciendo los conceptos de DGP-PD y DGP-PFI. Estas medidas representan los análogos poblacionales de las técnicas de interpretación, definidos en términos de la verdadera esperanza condicional $f(x)=E[Y|X=x]$:

\begin{equation}
\text{DGP-PD}(x_S)=E_{X_C}\left[f(x_S, X_C)\right]
\end{equation}

\begin{equation}
\text{DGP-PFI} = E_{\bar{X}_S, X_C, Y}\left[L\left(Y, f(\bar{X}_S, X_C)\right)\right] - E_{X,Y}\left[L\left(Y, f(X)\right)\right]    
\end{equation}

donde $\bar{X}_S$ representa una copia independiente de $X_S$, utilizada para simular la permutación.

Los autores descomponen el error de estimación de PDP y PFI en componentes de sesgo y varianza, identificando tres fuentes principales de error: (1) la aproximación de Monte Carlo al usar una muestra finita para calcular las esperanzas, (2) la posible especificación incorrecta del modelo (diferencia entre $\hat{f}$ y $f$), y (3) la variabilidad inherente al proceso de entrenamiento del modelo.

Para cuantificar la incertidumbre debida a la variabilidad del entrenamiento, proponen la metodología \textit{learner-PD/PFI}, que consiste en promediar los resultados de interpretación obtenidos a partir de múltiples modelos reentrenados con diferentes semillas o submuestras. Este enfoque permite construir intervalos de confianza que reflejan la incertidumbre asociada al proceso de aprendizaje.

No obstante, esta metodología tiene limitaciones. Asume que el modelo es aproximadamente insesgado, lo cual puede no cumplirse en la práctica. Además, cuando se utiliza \textit{bootstrap} para el reentrenamiento, el solapamiento entre las muestras puede generar intervalos de confianza demasiado estrechos. Finalmente, la dependencia entre variables complica la interpretación de las medidas, ya que tanto PDP como PFI asumen implícitamente independencia entre $X_S$ y $X_C$.
